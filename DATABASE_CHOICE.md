# Database Choice: Azure Cosmos DB (Serverless)

## My Choice

I selected Azure Cosmos DB (serverless tier) as the database for storing Text Analyzer results.

## Justification

Azure Cosmos DB is a fully managed NoSQL document database that natively supports JSON data, which matches the structure of the text analysis results generated by the Azure Function. 

Its serverless tier allows automatic scaling and cost-efficient usage, which is ideal for a student project with unpredictable workloads. It also integrates well with Azure Functions through the Python SDK, making it easy to store and retrieve analysis results. 

Compared to relational or table-based storage, Cosmos DB requires no schema design and provides more flexible querying capabilities.

## Alternatives Considered

Azure SQL Database was considered, but it requires a predefined schema and is less natural for storing dynamic JSON documents. Azure Table Storage was also considered but was rejected due to its limited querying capabilities and poor support for complex JSON structures. Azure Blob Storage can store JSON files, but it lacks built-in query functionality, making it unsuitable for retrieving and analyzing historical results efficiently.

## Cost Considerations

The serverless tier of Azure Cosmos DB charges based on request units (RU) and storage usage, with a free tier available for low-volume workloads. This makes it cost-effective for development and academic projects, as there is no need to provision dedicated throughput or continuously running infrastructure.
